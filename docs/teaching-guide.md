# Teaching Guide — Software Testing Lab

## Overview

This guide helps instructors deliver the Software Testing Lab course effectively. It covers session structure, learning objectives, pacing, and tips for common student challenges.

---

## Learning Objectives

By the end of this lab, students should be able to:

1. Explain the purpose and types of software testing.
2. Write unit tests using a modern testing framework.
3. Identify bugs using black-box and white-box techniques.
4. Understand CI/CD pipelines and automated test runs.
5. Use GitHub for version control and collaboration.

---

## Suggested Session Plan (8 Weeks)

| Week | Topic | Activities |
|------|-------|------------|
| 1 | Introduction to Testing | Lecture, flashcard review |
| 2 | Unit Testing Basics | Hands-on coding exercise |
| 3 | Integration Testing | Pair programming exercise |
| 4 | Test-Driven Development | TDD kata workshop |
| 5 | Black-box & White-box | Case study analysis |
| 6 | Regression & Smoke Testing | Automated test lab |
| 7 | CI/CD & GitHub Actions | Pipeline setup exercise |
| 8 | Final Project Presentations | Peer review & assessment |

---

## Tips for Instructors

- **Use the flashcards** at the start of each session for a 5-minute warm-up.
- **Encourage pair programming** — students learn faster when debugging together.
- **Focus on "why" before "how"** — students retain testing concepts better when they understand the motivation.
- For Week 7, ensure students have GitHub accounts set up in advance.
