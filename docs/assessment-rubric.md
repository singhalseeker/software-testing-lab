# Assessment Rubric — Software Testing Lab

## Grading Overview

| Component | Weight |
|-----------|--------|
| Weekly Exercises | 30% |
| Midterm Quiz | 20% |
| Final Project | 40% |
| Participation | 10% |

---

## Weekly Exercises Rubric

| Criteria | Excellent (4) | Good (3) | Satisfactory (2) | Needs Work (1) |
|----------|--------------|----------|-----------------|----------------|
| Correctness | All tests pass | Most tests pass | Some tests pass | Few/no tests pass |
| Code Quality | Clean, readable, well-structured | Minor style issues | Some readability issues | Hard to follow |
| Completeness | All tasks done | Most tasks done | Half tasks done | Minimal completion |
| On Time | Submitted on time | 1 day late | 2 days late | More than 2 days late |

---

## Final Project Rubric

### 1. Test Coverage (25 points)
- 25: >90% coverage, all edge cases handled
- 20: 75–90% coverage
- 15: 50–74% coverage
- 10: <50% coverage

### 2. Test Quality (25 points)
- 25: Tests are meaningful, isolated, and well-named
- 20: Most tests are meaningful
- 15: Tests exist but have issues
- 10: Tests are superficial or poorly designed

### 3. CI/CD Integration (20 points)
- 20: Pipeline runs automatically, all checks pass
- 15: Pipeline set up with minor issues
- 10: Partial setup
- 5: Attempted but not functional

### 4. Documentation (15 points)
- 15: Clear README, setup instructions, test descriptions
- 10: Adequate documentation
- 5: Minimal documentation

### 5. Presentation (15 points)
- 15: Clear explanation, answers questions confidently
- 10: Adequate presentation
- 5: Unclear or incomplete presentation
